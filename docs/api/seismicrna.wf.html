Run the entire workflow.

Parameters

  • out_dir (str) – Destination for all new output files [keyword-only, default: ‘./out’]

  • temp_dir (str) – Destination for all temporary files [keyword-only, default: ‘./temp’]

  • keep_temp (bool) – Keep temporary files after the program exits [keyword-only, default: False]

  • brotli_level (int) – Compression level for brotli (0 - 11) [keyword-only, default: 10]

  • force (bool) – Force all tasks to run, even those whose output files already exist [keyword-only, default: False]

  • max_procs (int) – Maximum number of simultaneous processes [keyword-only, default: 16]

  • parallel (bool) – Run tasks in parallel [keyword-only, default: True]

  • fastqz (tuple) – FASTQ files of single-end reads [keyword-only, default: ()]

  • fastqy (tuple) – FASTQ files of paired-end reads interleaved in 1 file [keyword-only, default: ()]

  • fastqx (tuple) – FASTQ files of paired-end reads separated into 2 files [keyword-only, default: ()]

  • phred_enc (int) – Phred score encoding in FASTQ and SAM/BAM/CRAM files [keyword-only, default: 33]

  • demulti_overwrite (bool) – desiginates whether to overwrite the grepped fastq. should only be used if changing setting on the same sample [keyword-only, default: False]

  • demult_on (bool) – Enable demultiplexing [keyword-only, default: False]

  • parallel_demultiplexing (bool) – Whether to run demultiplexing at maximum speed by submitting multithreaded grep functions [keyword-only, default: False]

  • clipped (int) – Designates the amount of clipped patterns to search for in the sample, will raise compution time [keyword-only, default: 0]

  • mismatch_tolerence (int) – Designates the allowable amount of mismatches allowed in a string and still be considered a valid pattern find. will increase non-parallel computation at a factorial rate. use caution going above 2 mismatches. does not apply to clipped sequences. [keyword-only, default: 0]

  • index_tolerance (int) – Designates the allowable amount of distance you allow the pattern to be found in a read from the reference index [keyword-only, default: 0]

  • barcode_start (int) – index of start of barcode [keyword-only, default: 0]

  • barcode_end (int) – length of barcode [keyword-only, default: 0]

  • dmfastqz (tuple) – Demultiplexed FASTQ files of single-end reads [keyword-only, default: ()]

  • dmfastqy (tuple) – Demultiplexed FASTQ files of paired-end reads interleaved in one file [keyword-only, default: ()]

  • dmfastqx (tuple) – Demultiplexed FASTQ files of mate 1 and mate 2 reads [keyword-only, default: ()]

  • fastqc (bool) – Run FastQC on the initial and trimmed FASTQ files [keyword-only, default: True]

  • qc_extract (bool) – Unzip FastQC report files [keyword-only, default: False]

  • cut (bool) – Trim reads with Cutadapt before alignment [keyword-only, default: True]

  • cut_q1 (int) – Phred score for read 1 quality trimming with Cutadapt [keyword-only, default: 25]

  • cut_q2 (int) – Phred score for read 2 quality trimming with Cutadapt [keyword-only, default: 25]

  • cut_g1 (tuple) – 5’ adapter for read 1 adapter trimming with Cutadapt [keyword-only, default: (‘GCTCTTCCGATCT’,)]

  • cut_a1 (tuple) – 3’ adapter for read 1 adapter trimming with Cutadapt [keyword-only, default: (‘AGATCGGAAGAGC’,)]

  • cut_g2 (tuple) – 5’ adapter for read 2 adapter trimming with Cutadapt [keyword-only, default: (‘GCTCTTCCGATCT’,)]

  • cut_a2 (tuple) – 3’ adapter for read 2 adapter trimming with Cutadapt [keyword-only, default: (‘AGATCGGAAGAGC’,)]

  • cut_o (int) – Minimum overlap of read and adapter during trimming with Cutadapt [keyword-only, default: 6]

  • cut_e (float) – Error tolerance for adapters during trimming with Cutadapt [keyword-only, default: 0.1]

  • cut_indels (bool) – Allow indels in adapters during trimming with Cutadapt [keyword-only, default: True]

  • cut_nextseq (bool) – Trim high-quality Gs from 3’ end during trimming with Cutadapt [keyword-only, default: False]

  • cut_discard_trimmed (bool) – Discard reads in which an adapter was found by Cutadapt [keyword-only, default: False]

  • cut_discard_untrimmed (bool) – Discard reads in which no adapter was found by Cutadapt [keyword-only, default: False]

  • cut_m (int) – Minimum length of a read to keep it after trimming with Cutadapt [keyword-only, default: 20]

  • bt2_local (bool) – Run Bowtie2 in local mode [keyword-only, default: True]

  • bt2_discordant (bool) – Output discordant alignments from Bowtie2 [keyword-only, default: False]

  • bt2_mixed (bool) – Attempt to align individual mates of unaligned pairs with Bowtie2 [keyword-only, default: False]

  • bt2_dovetail (bool) – Treat dovetailed mate pairs as concordant with Bowtie2 [keyword-only, default: False]

  • bt2_contain (bool) – Treat nested mate pairs as concordant with Bowtie2 [keyword-only, default: True]

  • bt2_score_min_e2e (str) – Minimum score for a valid alignment with Bowtie2 in end-to-end mode [keyword-only, default: ‘L,-1,-0.5’]

  • bt2_score_min_loc (str) – Minimum score for a valid alignment with Bowtie2 in local mode [keyword-only, default: ‘L,1,0.5’]

  • bt2_i (int) – Minimum fragment length for valid paired-end alignments with Bowtie2 [keyword-only, default: 0]

  • bt2_x (int) – Maximum fragment length for valid paired-end alignments with Bowtie2 [keyword-only, default: 600]

  • bt2_gbar (int) – Minimum distance of an indel from the end of a read with Bowtie2 [keyword-only, default: 4]

  • bt2_l (int) – Seed length for Bowtie2 [keyword-only, default: 20]

  • bt2_s (str) – Seed interval for Bowtie2 [keyword-only, default: ‘L,1,0.1’]

  • bt2_d (int) – Maximum number of consecutive failed seed extensions with Bowtie2 [keyword-only, default: 4]

  • bt2_r (int) – Maximum number of re-seeding attempts with Bowtie2 [keyword-only, default: 2]

  • bt2_dpad (int) – Width of padding on alignment matrix (to allow indels) with Bowtie2 [keyword-only, default: 2]

  • bt2_orient (str) – Valid orientations of paired-end mates with Bowtie2 [keyword-only, default: ‘fr’]

  • bt2_un (bool) – Output unaligned reads from Bowtie2 to a FASTQ file [keyword-only, default: True]

  • min_mapq (int) – Minimum mapping quality to use an aligned read [keyword-only, default: 25]

  • cram (bool) – Compress alignment maps using CRAM format [keyword-only, default: False]

  • min_phred (int) – Minimum Phred score to use a base call [keyword-only, default: 25]

  • min_reads (int) – Minimum number of reads in an alignment map [keyword-only, default: 1000]

  • ambrel (bool) – Mark all ambiguous indels [keyword-only, default: True]

  • overhangs (bool) – Compute relationships for bases in a read that extend beyond its mate [keyword-only, default: True]

  • batch_size (float) – Target size of each batch (megabases) [keyword-only, default: 64.0]

  • pool (str) – Name of the sample made by pooling other samples [keyword-only, default: ‘’]

  • coords (tuple) – Reference name and 5’/3’ ends of a section; ends are 1-indexed [keyword-only, default: ()]

  • primers (tuple) – Reference name and forward/reverse primers for a section; for the reverse primer, use its actual sequence, not its reverse complement [keyword-only, default: ()]

  • primer_gap (int) – Length of the gap (nt) between each primer and the end of the section [keyword-only, default: 0]

  • sections_file (str) – CSV file of sections by name, reference, and coordinates/primers [keyword-only, default: ‘’]

  • count_del (bool) – Count deletions as mutations [keyword-only, default: False]

  • count_ins (bool) – Count insertions as mutations [keyword-only, default: False]

  • discount_mut (tuple) – Types of mutations to ignore [keyword-only, default: ()]

  • exclude_polya (int) – Minimum length of poly(A) sequences to exclude (0 to include all) [keyword-only, default: 5]

  • exclude_gu (bool) – Exclude G and U bases [keyword-only, default: True]

  • exclude_file (str | None) – File of arbitrary positions to exclude [keyword-only]

  • min_ncov_read (int) – Minimum coverage of a read to keep it (nt) [keyword-only, default: 1]

  • min_finfo_read (float) – Minimum fraction of information in a read to keep it [keyword-only, default: 0.95]

  • max_fmut_read (int) – Maximum fraction of mutations in a read to keep it [keyword-only, default: 0.1]

  • min_mut_gap (int) – Minimum gap between two mutations in a read to keep it [keyword-only, default: 3]

  • min_ninfo_pos (int) – Minimum information count at a position to use it [keyword-only, default: 1000]

  • max_fmut_pos (float) – Maximum mutation fraction at a position to use it [keyword-only, default: 0.5]

  • max_clusters (int) – Maximum number of clusters to attempt (0 to disable) [keyword-only, default: 0]

  • em_runs (int) – Number of independent runs for each number of clusters [keyword-only, default: 6]

  • min_em_iter (int) – Minimum iterations per clustering run [keyword-only, default: 8]

  • max_em_iter (int) – Maximum iterations per clustering run [keyword-only, default: 512]

  • em_thresh (float) – Maximum change in log likelihood for convergence [keyword-only, default: 0.1]

  • joined (str) – Name of the section made by joining other sections [keyword-only, default: ‘’]

  • join_clusts (str) – CSV file of clusters to join [keyword-only]

  • table_pos (bool) – Tabulate per position [keyword-only, default: True]

  • table_read (bool) – Tabulate per read [keyword-only, default: True]

  • table_clust (bool) – Tabulate per cluster [keyword-only, default: True]

  • fold (bool) – Predict the secondary structure using the RNAstructure Fold program [keyword-only, default: False]

  • quantile (float) – Quantile for normalizing ratios; must be in [0, 1] [keyword-only, default: 0.0]

  • fold_temp (float) – Temperature at which to predict structures (in Kelvin) [keyword-only, default: 310.15]

  • fold_constraint (str | None) – File of constraints for predicting structures [keyword-only]

  • fold_md (int) – Maximum distance between two paired bases in predicted structures (use 0 for no limit) [keyword-only, default: 0]

  • fold_mfe (bool) – Predict only the minimum free energy (MFE) structure, without any suboptimal structures [keyword-only, default: False]

  • fold_max (int) – Maximum number of predicted structures [keyword-only, default: 20]

  • fold_percent (float) – Maximum % difference in energy between suboptimal structures [keyword-only, default: 20.0]

  • export (bool) – Export per-sample results for the seismic-graph web app [keyword-only, default: False]

  • samples_meta (str) – CSV file of metadata for each sample [keyword-only]

  • refs_meta (str) – CSV file of metadata for each reference [keyword-only]

  • all_pos (bool) – Export all positions (not just unmasked positions) [keyword-only, default: False]

  • cgroup (str) – Graph each INDIVidual cluster in its own file, each ORDER in its own file, or UNITE all clusters in one file containing all orders [keyword-only, default: ‘order’]

  • hist_bins (int) – Number of bins in each histogram (≥ 1) [keyword-only, default: 10]

  • hist_margin (float) – For ratio-based histograms, maximum margin to autofill between the data and 0 or 1 [keyword-only, default: 0.01]

  • struct_file (str) – CT file of structures, against which to compare mutational profiles [keyword-only]

  • struct_sect (str) – Section of RNA that was folded, against which to compare mutational profiles (ignored if blank or if using –struct-file) [keyword-only, default: ‘’]

  • window (int) – Size of the sliding window, in nucleotides [keyword-only, default: 45]

  • winmin (int) – Minimum number of data to use a sliding window (otherwise NaN) [keyword-only, default: 9]

  • csv (bool) – Output the source data for each graph as a CSV file [keyword-only, default: True]

  • html (bool) – Output each graph as an HTML file [keyword-only, default: True]

  • pdf (bool) – Output each graph as a PDF file [keyword-only, default: False]
