seismicrna.graph.seqbar.run(input_path: tuple[str, ...], rels: tuple[str, ...] = ('m',), *, y_ratio: bool = True, quantile: float = 0.0, arrange: str = 'order', csv: bool = True, html: bool = True, pdf: bool = False, force: bool = False, max_procs: int = 16) → list[pathlib.Path]

Run the graph seqbar module.

Parameters

  • rels (tuple) – Relationships to graph [positional or keyword, default: (‘m’,)]

  • y_ratio (bool) – Graph counts or ratios on the y-axis [keyword-only, default: True]

  • quantile (float) – Quantile of mutation rates for normalization; must be in [0, 1] [keyword-only, default: 0.0]

  • arrange (str) – Graph each INDIVidual cluster in its own file, each ORDER in its own file, or UNITE all clusters in one file containing all orders [keyword-only, default: ‘order’]

  • csv (bool) – Output the source data for each graph as a CSV file [keyword-only, default: True]

  • html (bool) – Output each graph as an HTML file [keyword-only, default: True]

  • pdf (bool) – Output each graph as a PDF file [keyword-only, default: False]

  • force (bool) – Force all tasks to run, even those whose output files already exist [keyword-only, default: False]

  • max_procs (int) – Maximum number of simultaneous processes [keyword-only, default: 16]
